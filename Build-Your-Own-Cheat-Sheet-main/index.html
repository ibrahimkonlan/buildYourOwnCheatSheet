SEMANTIC HTML

Semantic elements = elements with a meaning.

What are Semantic Elements?

A semantic element clearly describes its meaning to both the browser and the developer.
Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.
Examples of semantic elements: <form>, <table>, and <article> - Clearly defines its content.
In HTML there are some semantic elements that can be used to define different parts of a web page:

  • <article>
  • <aside>
  • <details>
  • <figcaption>
  • <figure>
  • <footer>
  • <header>
  • <main>
  • <mark>
  • <nav>
  • <section>
  • <summary>
  • <time>

SEMANTIC TAGS REFERENCE

Tag Name Description
<section> Section The <section> element defines a section in a document.
<article> Article The <article> element specifies independent, self-contained content.
<header> Header The <header> element represents a container for introductory content or a set of navigational links.
<footer> Footer The <footer> element defines a footer for a document or section.
<aside> Aside The <aside> element defines some content aside from the content it is placed in (like a sidebar).
<nav> Navigation The <nav> element defines a set of navigation links.
<figure> Figure The <figure> tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.
<figurecaption> Figure Caption The <figurecaption> tag defines a caption for a figure element. The figcaption element can be placed as the first or as the last child of a figure element.
